# Play Console Form-By-Form Checklist

## 1. App Details

- App name: `Match A Man`
- Default language: Turkish
- App type: App
- Free/paid decision: choose before creating the release
- Category: sports/social/event category that best matches current Play Console
  options
- Contact email: use the production support contact

## 2. Store Listing

- App title
- Short description
- Full description
- App icon
- Feature graphic
- Phone screenshots
- No fake/private data in screenshots
- No exaggerated safety claims
- Not positioned as dating-only

## 3. App Content

- Privacy Policy URL
- App access
- Ads declaration
- Content rating
- Target audience
- News app declaration if asked
- Data Safety
- Government apps declaration if asked
- Financial features declaration if asked

## 4. Data Safety

Verify against `docs/play_store_data_safety_draft.md`:

- Auth/email/profile data
- User-generated content
- Photos/images if uploaded
- Location/event location if used
- Messages/chat if implemented
- Feedback/reports
- Service providers/Supabase
- Data deletion path
- Security practices

## 5. App Access

- Login is required for main features.
- Provide reviewer test account privately in Play Console only.
- Do not commit real credentials.
- Reviewer account should be email-confirmed.
- Reviewer should be able to access Home, Events, Profile, and Settings.

## 6. Closed Testing

- Create closed test track.
- Upload signed AAB.
- Add testers via email list or Google Group.
- Add release notes.
- Review warnings.
- Roll out to closed testing.
- Copy opt-in link.
- Send opt-in link privately to testers.
