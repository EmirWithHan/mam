# Match A Man Demo Script

## Demo Goal

This demo shows Match A Man as an event-centered social sports app.

- Events are the core product loop.
- MaM is not a dating app.
- Users move through a real-world event flow: create, request, approve, chat, and coordinate.
- Feed posts add social proof around sports activity.
- Safety is supported through profile completion, host approval, reports, blocks, and trust score.

## Demo Accounts

Use at least two test users:

- User A, event host: `host@example.com`
- User B, participant: `participant@example.com`

Do not use real passwords in the script, slides, recordings, or shared notes.

## Demo Flow

1. Open the app as a logged-out user.
2. Register or log in as User A.
3. Complete User A's profile.
4. Upload an avatar if available.
5. Create an event.
6. Show the event in the Events list.
7. Log out.
8. Register or log in as User B.
9. Complete User B's profile.
10. Open Events.
11. Open the event detail page.
12. Request to join the event.
13. Show the pending request status.
14. Log out.
15. Log in as User A.
16. Open the event detail page.
17. Approve User B.
18. Open the event chat.
19. Send a message.
20. Log out.
21. Log in as User B.
22. Open the Social tab.
23. Open the event chat group.
24. Reply to the chat.
25. Use the call host button if safe to demo.
26. Create a feed post.
27. Demonstrate like, comment, and follow.
28. Demonstrate report/block only if appropriate for the audience.
29. Quickly show profile, trust score, and settings.

## Talk Track

- "Match A Man starts with real sports and social events, not romantic matching."
- "The feed supports the event loop by showing moments from the community."
- "Users do not automatically get full access. Hosts approve participants before chat and calls become useful."
- "Privacy matters: phone numbers are not public profile content, and call access is controlled by event participation."
- "Trust and safety are built into the MVP through profile completion, approval, reporting, blocking, and trust score signals."

## Demo Warnings

- Use test accounts only.
- Do not show real phone numbers on screen if avoidable.
- Confirm Supabase email confirmation settings before the demo.
- Make sure the demo environment has at least one event and one post before presenting.
- Avoid demonstrating report/block on real users or real content.
