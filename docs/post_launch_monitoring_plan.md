# Post-Launch Monitoring Plan

## Day 0

- Confirm app live.
- Test production install.
- Test signup/verification.
- Test username onboarding.
- Check core screens.
- Watch crash/user feedback.

## Day 1

- Collect reports.
- Classify BLOCKER/HIGH/MEDIUM/LOW.
- Fix only urgent issues.

## Days 2-3

- Monitor auth/onboarding/events/profile.
- Update FAQ/support answers.

## Days 4-7

- Decide first patch.
- Prepare versionCode/build number bump if patch needed.
- Avoid adding new features unless blocker.

## Manual Metrics To Watch

- Install/open issues.
- Signup completion.
- Email verification problems.
- Onboarding drop-off reports.
- Event creation/join errors.
- Crashes/white screens.
- Store review feedback.
- Support messages.
