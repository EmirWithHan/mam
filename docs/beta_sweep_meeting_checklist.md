# Beta Sweep Meeting Checklist

Use this during Android APK/closed testing and iOS TestFlight.

## Daily During First Tester Days

1. Collect all reports.
2. Classify severity using `docs/beta_bug_severity_rubric.md`.
3. Merge duplicates.
4. Ask for missing screenshot/video/logs.
5. Fix BLOCKER only.
6. Fix HIGH if core flow.
7. Defer MEDIUM/LOW unless very cheap.
8. Rebuild/retest.
9. Update `docs/beta_feedback_master_board.md`.

## Weekly Or Mid-Beta

1. Review unresolved BLOCKER/HIGH.
2. Decide if wider beta is safe.
3. Decide if Play closed test can continue.
4. Decide if TestFlight external beta is safe.
5. Update `docs/launch_candidate_report.md`.

## Meeting Output

- Updated board.
- Hotfix prompt for each BLOCKER.
- Deferred list for MEDIUM/LOW issues.
- Launch-candidate decision status.
