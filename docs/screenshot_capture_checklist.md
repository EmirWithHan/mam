# Screenshot Capture Checklist

Date: 2026-06-06

## Before Capture

- Use a release or near-release build.
- Confirm there is no debug banner.
- Do not show fake broken data.
- Do not show personal real user information.
- Do not show real phone/email values.
- Do not show raw error messages.
- Confirm there are no yellow/black overflow stripes.
- Confirm there is no English/Turkish mixed UI on target screens.
- Do not show placeholder lorem ipsum.
- Do not show admin/debug-only screens.
- Use stable internet.
- Keep status bar clean if possible.
- Use a consistent device size for the screenshot set.

## Accounts

- Use beta-safe test accounts.
- Use non-personal usernames.
- Use staged posts/events.
- Avoid real private people's photos unless permission exists.

## Data

- At least 2-3 sample events.
- At least 2 sample posts.
- One clean profile.
- One event detail with proper date/location/capacity.
- No offensive content.
- No real address unless intended.

## After Capture

- Crop only if needed.
- Keep aspect ratio.
- Export PNG/JPEG.
- Place final assets later under `store_assets/android/screenshots/`.
- Do not commit huge raw captures unless intentionally needed.
