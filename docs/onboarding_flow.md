# Onboarding Flow

Date: 2026-06-06

## Current Beta Flow

1. Email/password users sign up.
2. Supabase sends the email confirmation link.
3. The user opens the confirmation link, then logs in or returns with a verified session.
4. The app creates or loads the user's own `profiles` row.
5. If `username` is missing, the app routes to `Kullanıcı adını seç`.
6. The user chooses a username.
7. The app saves the username on the user's own profile row.
8. If the display name/name field is empty or required by current app usage, it is temporarily set to the same value as username.
9. The user enters the main app.

Google/OAuth users follow the same minimum profile rule: if they have no username, they choose only a username before entering the app.

## Minimum Profile

Required for app access:

- Authenticated user id.
- Valid username.
- Existing tag generated by the app when missing.

Optional after first entry:

- Full name/display name beyond the username fallback.
- Photo/avatar.
- Bio.
- City and district.
- Gender.
- Birth date.
- Phone.
- Interests/social links.
- Business information.

## Full Profile Editing

Full profile completion is optional during beta. Users can open it later from:

- `Profil`
- `Profili düzenle`
- `Ayarlar`, when linked there

The optional profile screen can still show `Profilini tamamla`, but app access is not blocked once username is set.

## Existing Users

- Existing users with a username go directly to the app.
- Existing users without a username go to `Kullanıcı adını seç`.
- Existing users with a display name but no username still choose a username.
- Existing users with username but no display name can safely get `display_name/name = username` on the next profile save when that field is empty.
